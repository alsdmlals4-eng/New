# Contributing

`my little boat`는 작은 보트에서 바다를 감상하는 1인칭 힐링 표류 게임입니다. 모든 기여는 이 핵심 경험을 해치지 않는 방향이어야 합니다.

2026-07-09 기준으로 새로 기획하는 소규모 게임은 Godot 기준으로 진행합니다. 기존 `my little boat` Unity MVP는 보존하고, Unity 작업은 이 MVP를 직접 수정할 때만 사용합니다.

## 작업 전 확인

1. 관련 Issue가 있는지 확인합니다.
2. 없으면 먼저 Issue를 만듭니다.
3. 대상 엔진이 Godot인지 Unity인지 확인합니다.
4. 변경 범위를 작게 잡습니다.
5. Godot 프로토타입은 `godot-prototypes/` 아래에 둡니다.
6. Unity MVP를 수정할 때는 Unity에서 열 수 있는 프로젝트 구조를 유지합니다.

## 코드 원칙

- 초보자가 읽을 수 있는 명확한 이름을 사용합니다.
- 스크립트는 가능한 한 한 가지 책임만 갖게 합니다.
- Godot 작업은 GDScript와 단순한 씬 구조를 우선 사용합니다.
- Godot 씬 이름과 노드 이름은 역할을 알 수 있게 짓습니다.
- Unity 작업에서 `SerializeField`는 null 가능성을 고려합니다.
- 주요 public 메서드 또는 중요한 GDScript 함수에는 짧은 주석을 둡니다.
- Godot 또는 Unity에서 실행 에러가 없어야 합니다.
- 유료 에셋에 의존하지 않습니다.

## 게임 방향

추가해도 좋은 것:

- 마음 선택에 따른 색감, 소리, 이벤트 변화
- 사진, 편지, 풍경, 동반자 반응 수집
- 감상모드와 방치형 감상 경험 개선
- 모바일 세로 화면 UI 개선

추가하지 말아야 할 것:

- 전투
- 실패 조건
- 경쟁 점수
- 결제
- 광고
- 온라인 편지 공유
- 복잡한 상점

## 테스트

Pull Request 또는 커밋 전 아래를 확인합니다.

Godot 프로토타입:

- 대상 `.tscn` 씬이 실행되는가
- 모바일 세로 화면 기준 UI가 잘 보이는가
- PC 입력과 모바일 입력이 모두 고려되었는가
- 핵심 루프가 1분 안에 확인되는가
- 저장 기능이 있다면 로컬에서 정상 동작하는가

Unity `my little boat`:

- MainMenuScene에서 마음 선택 후 GameScene으로 이동하는가
- GameScene에서 플레이어 몸이 보이지 않는가
- 사진찍기, 감상모드, 속도조절이 작동하는가
- 편지 또는 풍경 이벤트가 발생하는가
- 앨범에 사진, 풍경, 편지가 기록되는가
- 모바일 세로 화면에서 주요 버튼이 잘 보이는가
