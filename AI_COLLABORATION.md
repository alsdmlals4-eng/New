# AI Collaboration Guide

이 저장소는 사람, GPT, Codex가 GitHub를 경유해 함께 작업하는 방식을 기준으로 운영합니다.

## 고정 기준

- 프로젝트명: `my little boat`
- 엔진: Godot 4.7 stable
- 언어: GDScript
- 장르: 1인칭 힐링 표류 게임
- 우선 플랫폼: 모바일 세로 화면, PC 마우스 입력
- 금지: 전투, 실패 조건, 경쟁, 결제, 광고, 온라인 편지 공유

## 역할

### 사람

- 최종 방향과 우선순위를 정합니다.
- 기능 추가, 버그 수정, 기획 변경을 Issue로 남깁니다.
- Godot에서 실행 테스트를 하고 최종 판단을 합니다.

### GPT

- 기획, UX, 밸런스, 문서, 테스트 관점에서 제안합니다.
- Issue를 만들 때 목적, 완료 기준, 사용자 경험을 명확히 정리합니다.
- Pull Request를 리뷰할 때 코드뿐 아니라 게임 콘셉트와 플레이 감정도 함께 봅니다.

### Codex

- Issue 또는 명확한 요청을 기준으로 Godot 프로젝트를 수정합니다.
- GDScript, 씬, 문서, 테스트 체크리스트를 함께 갱신합니다.
- 변경 내용을 커밋하고 GitHub에 푸시할 수 있습니다.

## 작업 흐름

1. 사람 또는 GPT가 Issue를 작성합니다.
2. Issue에는 목표, 배경, 완료 기준, 테스트 방법을 적습니다.
3. Codex가 해당 Issue를 기준으로 파일을 수정합니다.
4. 변경 사항은 Pull Request 또는 main 커밋으로 남깁니다.
5. 사람이 GitHub Desktop에서 Pull 후 Godot에서 테스트합니다.
6. GPT가 리뷰할 때는 PR 설명과 변경 파일을 기준으로 의견을 남깁니다.

## Issue 작성 원칙

- 한 Issue에는 한 가지 목표만 적습니다.
- 결과를 구체화합니다. 예: "감상모드 버튼을 누르면 HUD가 숨겨지고 다시 누르면 복구된다."
- Godot에서 확인할 테스트 방법을 반드시 적습니다.
- 제외할 것도 적습니다. 예: 결제, 광고, 온라인 기능 제외.

## Pull Request 작성 원칙

- 변경한 기능을 짧게 요약합니다.
- 확인한 Godot 버전과 실행한 씬을 적습니다.
- 아직 확인하지 못한 위험을 숨기지 않습니다.
- GPT 리뷰를 요청할 때는 "버그 가능성", "UX 어색함", "초보자 이해 가능성" 중 무엇을 봐야 하는지 적습니다.

## Codex에게 줄 수 있는 요청 예시

```text
Issue #3을 구현해줘.
Godot 4.7 stable에서 실행 가능하게 만들고 README 테스트 체크리스트도 갱신해줘.
```

```text
main_menu.tscn에 마음 선택 버튼을 더 보기 좋게 배치해줘.
모바일 세로 화면 기준으로 확인할 수 있게 해줘.
```
