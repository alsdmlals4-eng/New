# Godot Direction

이 문서는 2026-07-09 기준 엔진 방향 결정입니다.

## 결정

- 앞으로 새로 기획하는 소규모 게임은 Godot 기준으로 진행합니다.
- 현재 저장소의 `my little boat` Unity MVP는 보존하고, 참고용 프로토타입으로 유지합니다.
- Unity 작업은 `my little boat`를 계속 확장해야 할 때만 진행합니다.
- 비주얼노벨, 뱀서라이크, 타워디펜스는 Godot 4.x 기준으로 기획하고 구현합니다.

## Godot을 기본으로 쓰는 이유

- 2D 게임 제작 흐름이 가볍고 빠릅니다.
- UI, 대사, 선택지, 웨이브, 배치형 게임 구조를 작은 단위로 만들기 좋습니다.
- 프로젝트 파일이 비교적 단순해서 GitHub 협업에 유리합니다.
- 무료 오픈소스 엔진이라 소규모 실험을 반복하기 좋습니다.

## 기본 기술 방향

- 엔진: Godot 4.x 안정 버전
- 언어: GDScript 우선
- 화면: 모바일 세로 화면을 먼저 고려하고, PC 입력을 함께 지원
- 아트: 무료 또는 직접 제작한 에셋 우선
- 저장: 초기 MVP에서는 로컬 저장만 사용
- 배포: 초기에는 PC 또는 Web 테스트를 먼저 고려하고, 모바일 빌드는 후순위로 검토

## 저장소 사용 방식

현재 저장소 루트는 Unity MVP 구조입니다. Godot 프로토타입을 이 저장소 안에 둘 때는 루트에 바로 만들지 않고 아래 구조를 사용합니다.

```text
godot-prototypes/
  visual-novel-mvp/
    project.godot
  survivor-like-mvp/
    project.godot
  tower-defense-mvp/
    project.godot
```

나중에 하나의 게임을 본격 개발하게 되면, 그 게임은 별도 GitHub 저장소로 분리하는 것을 권장합니다.

## 작업 원칙

- 한 번에 하나의 작은 MVP만 만듭니다.
- 복잡한 프레임워크를 먼저 만들지 않습니다.
- 장르 핵심 재미가 확인되기 전까지 상점, 광고, 로그인, 온라인 기능은 제외합니다.
- 모든 Issue와 Pull Request에는 대상 엔진을 적습니다.
- GPT는 기획과 UX 흐름을 정리하고, Codex는 Godot 프로젝트 구현과 문서 갱신을 담당합니다.
